# UI_DESIGN_RULES.md

# UI Design Rules

## Design Philosophy

Priority:

1. Consistency
2. Readability
3. Efficiency
4. Simplicity

Avoid unnecessary visual complexity.

------

# Admin System Style

Reference:

- Stripe Dashboard
- Linear
- Notion
- Modern SaaS Admin

Use:

Element Plus

Avoid:

- Heavy Borders
- Excessive Colors
- Nested Dialogs
- Overcrowded Pages

------

# Page Layout Standard

Every list page:

Search Area

↓

Action Buttons

↓

Table

↓

Pagination

------

# Search Area

Use:

Reset
Search

Buttons.

Search conditions should be arranged horizontally.

Default:

1~2 rows maximum.

------

# Table Rules

Required:

Loading
Pagination
Empty State

Recommended:

Status Tag
Amount Formatter
Date Formatter

Sensitive Data Masking

------

# Dialog Rules

Simple Form:

600px

Normal Form:

800px

Detail View:

900px

Must support:

Validation
Loading
Submit Lock

------

# Form Rules

Required fields:

Display *

Use validation.

Avoid long forms in a single column.

Prefer:

Two-column layout

for complex forms.

------

# Status Display

Success:

Tag Style

Processing:

Tag Style

Failed:

Tag Style

Avoid plain text status display.

------

# Amount Display

Examples:

USD 1,234.56

EUR 2,999.00

Always align right in tables.

------

# Cashier Rules

Reference:

Stripe Checkout
Adyen Checkout

Design Principles:

- Mobile First
- Fast Loading
- Minimal Design
- Secure Appearance

Avoid:

Sidebar
Table
Complex Navigation

------

# Cashier Layout

Order Summary

↓

Payment Method

↓

Payment Form

↓

Pay Button

------

# Mobile Support

Must support:

375px width

Responsive Design Required.

------

# Accessibility

Buttons:

Minimum Height 40px

Inputs:

Minimum Height 40px

Interactive Elements:

Clearly Visible

------

# Color Usage

Success:

Green

Warning:

Orange

Error:

Red

Info:

Neutral

Do not use excessive colors.

------

# Typography

Prefer:

14px
16px

Avoid:

10px
11px

Text must remain readable.

------

# Empty State

Every page should provide:

Friendly Empty State

instead of blank tables.

------

# Error State

Display user-friendly messages.

Never expose:

SQL Error
Java Exception
Stack Trace

to end users.

------

# Final Goal

Every page generated by AI should look like:

Modern SaaS Product

instead of:

Traditional Enterprise Software.
